## How this DS is wired (read before re-syncing)
hisohiso's `app/` is a **private Vite app**, not a published component library â€” no
`main`/`module`/`exports`, ships no `.d.ts`. So the sync builds a small library surface:

- **Barrel entry**: `app/src/ds-entry.tsx` re-exports all 37 synced components under their
  real export names (default-only chrome components are aliased to named, e.g.
  `export { default as Avatar }`). The converter bundles THIS via `cfg.entry`, so app
  side-effect modules (`main.tsx`, pages) never enter the bundle. **Add/remove a synced
  component â†’ edit this barrel AND `cfg.componentSrcMap`.**
- **Types**: `app/package.json` has `"types": "ds-types/ds-entry.d.ts"`. `cfg.buildCmd` runs
  `tsc --emitDeclarationOnly` into `app/ds-types/` so the converter can extract real prop
  contracts. Without the `types` field, every component falls back to an empty
  `{ [key: string]: unknown }` body. **Do not remove the `types` field.**
- **tsc emits despite errors**: the app has a few pre-existing strictness errors (untyped
  `qrcode`, `crypto.ts` SharedArrayBuffer, two RoomController null-checks). tsc still emits
  all declarations, so `buildCmd` wraps tsc in `|| true`. These are NOT introduced by the sync.
- **CSS**: `cfg.buildCmd` compiles `app/src/styles.css` with `app/tailwind.ds.config.js`
  (Tailwind, same tokens as the app, content scan also covers `.design-sync/previews/`) into a
  STABLE path `app/ds-dist/ds-styles.css` (`cfg.cssEntry`). The app's own `dist/assets/*.css`
  has a content hash and is NOT used (would break on every app rebuild).
- **Fonts**: riso fonts live in `app/public/fonts/` but `riso-fonts.css` uses root-absolute
  `url(/fonts/â€¦)`, which the converter can't resolve â†’ `[FONT_DANGLING]`. `cfg.buildCmd`
  rewrites them to relative URLs into `app/ds-dist/fonts/ds-fonts.css` (+ copies the 12 woff2),
  and `cfg.extraFonts` points there.
- **Block contracts inlined**: the block `*BlockView` props reference domain types
  (`ButtonsBlockType`, etc.) that aren't defined in the emitted `.d.ts`. `cfg.dtsPropsFor`
  inlines the full block spec for each block (source of truth: `app/src/lib/blocks.ts`),
  giving self-contained contracts. **If `app/src/lib/blocks.ts` changes, update `dtsPropsFor`.**
- **Groups**: blocks â†’ `blocks` (from dir), chrome â†’ `general` (the `components/` dir is generic).

## Known render warns (triaged as legitimate â€” re-sync should not treat as new)
- `[FONT_MISSING] "IBM Plex Mono"` â€” it's only the *tertiary* fallback in the mono stack
  (`Space Mono`, `IBM Plex Mono`, `monospace`); Space Mono ships and is primary. Accepted the
  system-mono substitute for the never-reached fallback. Non-blocking.
- **ThinkingBlockView renders collapsed by default** (`open=false`) â€” a static capture shows
  only the disclosure pill (emoji + `summary` + caret); `content` is hidden behind a click.
  This is the component's genuine default. Always set a meaningful `block.summary` when
  composing it, or the pill reads generic. Correct to grade the collapsed pill as-is.
- **ConfirmDangerBlockView** capture reported "2 errors" once â€” benign captured console/text
  bleed (the âš  glyph + description string), not a render failure. The sheet renders correctly.

## Re-sync risks (what can silently go stale)
- `cfg.dtsPropsFor` is a hand-inlined mirror of `app/src/lib/blocks.ts`. A spec change there
  won't auto-propagate â€” re-check the block contracts after editing blocks.ts.
- `app/ds-types/` and `app/ds-dist/` are gitignored (regenerated by `cfg.buildCmd`). A fresh
  clone MUST run `buildCmd` before the converter, or `pkgJson.types` points at a missing file.
- The riso font filenames are content-hashed; if the fonts are re-subset upstream, the woff2
  names in `riso-fonts.css` change and `buildCmd`'s copy still works (globs `*.woff2`), but
  verify `[FONT_DANGLING]` stays clear.
- App-chrome components (RoomCard, RoomsRail, AppLock, etc.) are coupled to IndexedDB
  (Dexie) / react-router / app state. Some render only with a provider or mock data; any that
  can't render statically ship the floor card (honest baseline) â€” see config/chrome handling.
- Playwright pinned to `playwright@1.60.0` in `.ds-sync` to match the cached chromium-1223
  build (no download). A different cache build needs a matching playwright version.

## Chrome previews seed/spoof real browser state (tied to upstream â€” re-sync watch-list)
The 11 app-chrome previews are coupled to runtime behavior, so their `.tsx` files seed state
at module scope. If the upstream behavior changes, these previews silently break â€” re-check:
- **RoomsRail** (`previews/RoomsRail.tsx`): seeds `localStorage['hisohiso.rooms']` with a
  StoredRoom array, AND injects a scoped `<style>` to override `.rooms-rail`'s responsive
  `display:none` (<1024px) / `flex-direction:column` (lg+) so it renders in the 900px capture
  viewport. Set `nickname` on seeded rooms or control/agent cards read "unnamed channel". If
  the rail's responsive CSS or the `hisohiso.rooms` key/shape changes, update the seed/style.
- **AppLock** (`previews/AppLock.tsx`): seeds `localStorage['hisohiso.app_lock'] =
  {enabled:true, pin:{salt,hash}}` + clears `sessionStorage['hisohiso.app_unlocked']` to show
  the lock screen. Tied to `getAppLockConfig`/`isAppLockArmed`/`shouldStartLocked`.
- **InstallPrompt** (`previews/InstallPrompt.tsx`): spoofs `navigator.userAgent/platform/
  maxTouchPoints` to force `detectIOS()` true (headless has no `beforeinstallprompt`). The
  app icon `/icons/icon-192.png` 404s in the offline capture (broken-image glyph) â€” cosmetic,
  the live app serves it. Tied to `useInstallPrompt`'s `canInstall` gate.
- **ControlCommandBar / QrModal** (fixed bottom bar / `fixed inset-0` overlay): the preview
  wrapper uses inline `transform: translateZ(0)` + `position: relative` + `minHeight` to make
  the fixed element lay out inside the card. General recipe for any fixed/overlay component.
- **RoomCard** runs a Dexie/IndexedDB effect (`lastMessageMeta`) that throws a benign headless
  error; the card renders fully regardless. Expected, not a defect.
- All chrome use FIXED mock values (no Date.now/Math.random) so captures stay deterministic.

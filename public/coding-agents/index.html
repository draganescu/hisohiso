Private machines stay private.

Hisohiso starts as a quiet line between two people. The agent bridge uses exactly the same line. No account. No phone number. No contact graph. No workspace history. No chat company sitting between your repo and your phone.

  • Self-host the router. Run the message router yourself when the work is sensitive.
  • Or use ours. hisohiso.org sees ciphertext, the same as your own router would.
  • No accidental audience. No team graph. No workspace inbox. No forwarded thread. Just the one room.
  • The repo stays where it lives. The agent reaches your working tree, tests, git config, and credentials — none of that crosses the wire.

Type less. Tap more.

Slack and Telegram weren't built for an agent. The agent's best answer is often a diff, a passing test, a row of three buttons, or a careful yes/no. In a chat app, that becomes a wall of code blocks and you typing "yes" for the fourth time. We render those answers as touch.

  • The reply fits the question. Diffs as diffs. Logs as logs. Buttons as buttons. Danger confirmations as danger confirmations.
  • Multiple agents, one phone. One room each for Claude, Codex, Hermes, a local model, or a one-off task. They don't bleed into each other.
  • The code stays local. The agent runs where your working tree, tests, git config, and credentials already live.
  • Decisions are taps. Approve. Reject. Retry. Show diff. No keyboard required.

We don't replace your agent. We give it a phone.

If you want a native mobile app that manages providers, schedules, worktrees, voice, and PR loops, Paseo or Happy are the answer. We wrap the agent you already trust with a private line. That's the entire pitch.

  • Claude Code keeps Claude. We don't manage the login, the model, or the prompt.
  • The phone is not a dev machine. It is a steering wheel. Tap, approve, glance, nudge.
  • No workspace to administer. No team graph. No account directory. No shared inbox.
  • Native blocks, not Markdown. A button is a button. A diff is a diff. Not a code fence.

Hisohiso + Hermes

Talk to Hermes from an encrypted room.

Keep Hermes on your machine. Use hisohiso on your phone as the end-to-end encrypted remote control surface.

Get the Hermes setup skill CLI source

Why this exists

Terminal agents are powerful because they run near your files, tools, shells, and local credentials. Phone chat is convenient because it is always with you. Hisohiso connects those two without turning your agent session into a cloud account or a public bot.

The shape

Local command

The hisohiso daemon invokes a command on your laptop. Hermes still runs where your terminal runs.

Per-room memory

Each spawned Hermes room gets its own Hermes session id keyed by the encrypted room hash.

Mobile UI blocks

Hermes can return JSON blocks for diffs, terminals, progress, buttons, and errors that render on a phone.

Install the setup skill

Give Hermes the public skill URL, then ask it to set itself up for hisohiso. The skill creates a wrapper, registers it with the hisohiso daemon, and keeps room sessions isolated. 

hermes skills install https://hisohiso.org/skills/hermes.md
hermes -s hisohiso-hermes-bridge

Or open Hermes and say: install the skill at https://hisohiso.org/skills/hermes.md and use it to set up hisohiso.

What the skill configures

Installs or updates the hisohiso CLI

Uses the official installer and verifies the local `hisohiso` command.

Creates `hisohiso-hermes`

A shell wrapper around `hermes chat` that persists one Hermes session id per hisohiso room.

Registers a daemon agent named `hermes`

Send `hermes` in the control room to spawn a new encrypted room backed by local Hermes.

Enables phone-native output

The wrapper preloads a small mobile UI skill so Hermes can emit hisohiso block envelopes.

What you get in the room

Plain text when the answer is simple. Structured cards when the work is complex.

diffs terminal output progress file trees errors buttons danger confirms

Hermes stays local. Hisohiso carries encrypted messages. The server sees ciphertext.

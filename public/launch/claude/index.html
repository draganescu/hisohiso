Claude Code,
with the door closed.

Claude already has ways to chat. Hisohiso is for the local CLI case: Claude Code runs beside your repo, while your phone gets a private, touch-friendly steering channel instead of a Slack thread, Telegram bot, WhatsApp workaround, or full agent-cockpit detour.

Run with hisohiso All coding agents

Why use hisohiso here?

  • Private by default. No account, phone number, contact graph, team workspace, or readable server-side message history sits around the Claude session.
  • Self-hostable when it matters. You can run your own message router; if you use hisohiso.org, the channel encryption is still active.
  • Touch-friendly output. Claude can send diffs, command logs, choices, and confirmations as phone interactions instead of chat paste.
  • Local work stays local. Claude can still use the repo, shell, tests, and git state on your machine.

Use this for the line, not the cockpit.

Hisohiso does not try to out-feature Claude-oriented apps with worktree management, schedules, provider switching, voice, or PR dashboards. It wraps Claude Code with a private encrypted channel for the moments when you need to steer, approve, inspect, or interrupt from a phone.

Run it through a channel.

Install and sign in to Claude Code first. Then ask hisohiso to wrap it.

npm install -g @anthropic-ai/claude-code
claude
hisohiso wrap claude

The first two commands are Claude's setup. The last one creates the hisohiso channel around that local CLI.

What shows up on the phone.

You can ask for work in normal language. When Claude needs a decision, hisohiso can show a button, a diff, or a confirmation instead of burying everything in a chat transcript.

The point is not to replace review. It is to make the small interruptions readable when you are away from the keyboard.

claude · repo
fix the failing auth test
diff · auth.test.ts
- expect(token).toBe(null)
+ expect(token).toBeDefined()

Claude handles the code. Hisohiso keeps the line clean.

Use it when a normal messaging app is too broad and a phone app is too far from the repo.

Codex stays
where the git repo is.

Codex already gives you an agent. Hisohiso gives that local session a private, touch-friendly phone channel, so you can steer repo work without turning it into a Slack thread, Telegram bot, WhatsApp chat, vendor-app detour, or mobile copy-paste exercise.

Run with hisohiso All coding agents

Why use hisohiso here?

  • Keep Codex private. No account, phone number, contact graph, team workspace, or readable server-side message history sits around the session.
  • Keep Codex local. The CLI can use the repo, tests, shell, and git state on the machine where the project already lives.
  • Keep chat apps out of it. Slack, Telegram, and WhatsApp are built for people. They are not great places to inspect diffs, choose from options, or approve commands.
  • Keep the phone useful. You can tap decisions, check results, and nudge the loop without pretending the phone is a development environment.

Use this for the line, not the cockpit.

Hisohiso does not try to out-feature Codex-oriented apps with worktree management, schedules, provider switching, voice, or PR dashboards. It wraps Codex with a private encrypted channel for the moments when you need to steer, approve, inspect, or interrupt from a phone.

Run it through a channel.

Install and sign in to Codex first. Then wrap it with hisohiso from inside a git repository.

npm install -g @openai/codex
codex
hisohiso wrap codex

Codex expects to work in a git repo. Hisohiso does not change that; it only gives the session an encrypted channel.

What shows up on the phone.

The phone is for steering: describe the task, answer questions, and inspect the output. The actual file edits and commands still happen on the machine running Codex.

That keeps the workflow understandable. Hisohiso carries the conversation. Git carries the truth.

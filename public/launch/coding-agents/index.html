coding agents

Your agent shouldn't
shout through someone else's app.

You can already control coding agents from polished apps and full cockpits. Hisohiso is deliberately narrower: the agent runs beside your repo, the phone gets touch-friendly controls, and the conversation travels through an accountless encrypted channel instead of another workspace or vendor thread.

The privacy comes built in.

Hisohiso starts as a private channel for people: no account, no phone number, no contact graph, no forwarding network, no readable server-side history. That same encrypted channel is what the agent bridge uses.

  • You can self-host the router. Run the message router yourself when the work is sensitive.
  • The default router still sees ciphertext. If you use hisohiso.org, the end-to-end encryption is the same.
  • The conversation is not social. No contact list, workspace, team channel, forwarded thread, or accidental audience.
  • Private machines stay private. A local agent can work near local files, credentials, tests, and private models without moving the conversation into a chat company.

An answer isn't always a sentence.

Slack, WhatsApp, and Telegram are chat apps. They do not know that an agent answer might be a diff, a terminal log, a carousel of choices, or a yes/no approval. Hisohiso replies can be touch-optimized interactions instead of walls of text you have to answer by typing.

  • Output fits the task. Diffs, command output, buttons, carousels, and danger confirmations can become phone-friendly blocks.
  • Multiple agents are natural. One phone can hold separate channels for Claude, Codex, Hermes, a local model, or a one-off task.
  • The code stays local. The agent runs where your working tree, tests, git config, and credentials already live.
  • You type less. Common decisions can become taps instead of mobile keyboard work.

Not a coding-agent cockpit.

If you want a native mobile app that manages providers, worktrees, schedules, voice, GitHub PR loops, and many agent sessions, products like Paseo or Happy are the direct comparison. Hisohiso should win only when the channel itself is the product: private, accountless, self-hostable, readable, and good at small touch decisions.

  • It does not replace Claude Code or Codex. Those tools still do the coding and own their login, model behavior, and repo workflow.
  • It does not pretend the phone is a dev machine. The phone is for steering, approving, reading diffs, and checking output.
  • It avoids social/workspace assumptions. No team graph, account directory, shared workspace history, or accidental channel audience.
  • It keeps blocks visible. Buttons, diffs, terminal output, confirmations, and carousels are native phone surfaces, not Markdown decoration.

Start with the hisohiso CLI.

Install hisohiso, then wrap the coding agent you already use. Hisohiso does not install Claude or Codex for you, and it does not manage their login. It only carries the encrypted channel messages.

curl -fsSL https://raw.githubusercontent.com/draganescu/hisohiso/main/cli/install.sh | sh

We keep the line private. You keep the judgment.

Hisohiso keeps the channel private from the server. It does not make code changes correct. Keep using the boring habits that already work: small diffs, tests, git status, and real review before you ship anything.

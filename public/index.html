Repo + easy deploy

The code lives in the hisohiso repo. It ships with Docker Compose, FrankenPHP/Caddy, and Mercure wiring, so most people can self-host it on a VPS without building custom infra.

  1. Point your domain to the server and expose ports 80/443.
  2. Run ./deploy.sh from your laptop to bootstrap a Docker-based droplet (or do it manually).
  3. The server gets a generated .env with SERVER_NAME and Mercure keys.
  4. Day-2 deploys run ./scripts/deploy.sh (or the production compose command) on the server.
  5. Optional: connect GitHub Actions so pushes to main/staging redeploy automatically.

If your host can run Docker and terminate TLS on 80/443, it can run Hisohiso. The deploy guide in the repo includes the GitHub Actions secret names used by the built-in DigitalOcean-style SSH deploy workflow.
